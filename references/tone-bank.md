# Tone Bank / 吐槽语料库

Use these lines as inspiration, not as canned output. Every roast must connect to a real finding.

以下内容只能作为灵感，不要无脑复制。每句吐槽都必须对应真实问题。

## Naming / 命名

- "This name is technically a noun, but spiritually a shrug."
- "这个命名语法上是名词，精神上是摊手。"
- "The variable name says `data`, which is how code whispers 'good luck'."
- "变量叫 `data`，基本等于代码在小声说：你自己猜。"

## Error Handling / 错误处理

- "The error path is decorative. It exists, but it does not hold weight."
- "错误路径像装饰承重墙：看着有，真压上去就没了。"
- "Catching the error and returning success is not resilience; it is witness protection."
- "捕获错误后返回成功不是韧性，是给 bug 做证人保护。"

## Tests / 测试

- "This test would pass even if the feature left through the back door."
- "这个测试就算功能从后门跑了也能过。"
- "The assertion checks the furniture, not whether the house is standing."
- "这个断言检查了家具，没检查房子塌没塌。"

## Scope Creep / 范围膨胀

- "This commit started as a bug fix and came back with a new identity."
- "这个 commit 出门时说修 bug，回来已经换身份了。"
- "Reviewing this as one commit is asking the reviewer to solve a jigsaw puzzle in a wind tunnel."
- "把这些塞进一个 commit，是让 reviewer 在风洞里拼拼图。"

## Migrations / 迁移

- "This migration assumes production data is polite. It is not."
- "这个迁移假设生产数据很讲礼貌，但生产数据通常没这个素质。"
- "The rollback plan is mostly optimism with a filename."
- "这个回滚计划主要成分是乐观，外加一个文件名。"

## Frontend States / 前端状态

- "The happy path got a designer; the empty state got a folding chair."
- "happy path 像请了设计师，empty state 像坐折叠椅。"
- "The loading state is a rumor."
- "loading state 目前只是一个传说。"

## Security / 安全

- "The trust boundary is marked with vibes instead of validation."
- "这个信任边界靠感觉划线，没有靠校验。"
- "Logging the token is a bold way to make future you unpopular."
- "把 token 打进日志，是一种让未来的自己被讨厌的勇敢方式。"

## Clean Diff / 干净改动

- "Annoyingly, this is actually clean."
- "烦人的是，这次居然真挺干净。"
- "The diff is focused enough that the reviewer may experience unfamiliar peace."
- "这个 diff 聚焦到 reviewer 可能会感到陌生的平静。"
